MODULE CONTEXT:

- selection of metrics (free to choose):
	- CPU utilisation - (used MIPS/total MIPS)
	- CPU demand pressure - (requested MIPS/total MIPS)
	- VM Count (number of VMs deployed)
- Data structure input: N/A
- Data structure output: array of double `double[]`

NAMING CONVENTION: "monitor_vX.java", where X >= 1 
## GUID NAMING CONVENTIONS

- No input so no input GUID.
- Output GUID must be a string and take the form `host-<metric>-<type>`, where metric is either "cpu" or "vm" and type is the nature of the metric. For example, "util" (0 to 1), "demand" (may exceed 1), "count" (raw count). For example "host-cpu-util", for CPU utilisation.
